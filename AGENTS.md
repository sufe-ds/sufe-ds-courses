# AGENTS.md

这份文档是给来仓库里找资料、补资料的同学看的。

## 先看哪里

1. 先读根目录的 `README.md`，了解仓库整体分区。
2. 找到对应课程后，再读该课程目录下已有的 `README.md`。
3. 如果想快速判断一个目录里有什么，优先看目录名和文件名，通常已经足够。

## 仓库怎么分

- `课程资料/`：课件、作业、试卷、笔记、参考资料。
- `密谈/`：回忆版试卷、选课指南、整理后的经验材料等。

如果你不确定该放哪，一般先放到最贴近课程的目录，再在 PR 里说明。

## 资料整理建议

- 文件名尽量清楚，写上课程名、学期、章节或用途。
- 新建目录时，沿用现有命名风格，别突然换一套。
- 资料里如果有个人信息，先脱敏再提交。
- 尽量不要提交缓存、临时文件、编辑器私有文件或编译产物。
- 如果是扫描件、截图或图片资料，尽量保证清晰可读。

## 贡献时怎么做

- 先建分支，再提交变更。
- 一次 PR 尽量只做一类内容，方便大家看。
- PR 标题直接写课程名和资料类型，比如“数据库期末复习资料”。
- PR 描述里写清楚：
  - 新增了什么
  - 放在哪个目录
  - 来源是什么
  - 是否做了脱敏

## 给后来的同学

这个仓库的目标很简单：让后来的人少走一点弯路。
如果你补了一份资料，顺手把目录、文件名、说明也整理一下，会比单纯多扔几个文件更有帮助。
