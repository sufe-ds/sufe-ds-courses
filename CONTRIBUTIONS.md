# CONTRIBUTIONS.md

欢迎补充课程资料。下面是推荐的标准 PR 流程。

## 1. 准备分支

先同步主分支，再新建一个自己的分支。

建议分支名：

- `feat/<course>-<topic>`
- `docs/<course>-<topic>`
- `fix/<course>-<topic>`

## 2. 添加资料

把资料放到最贴近课程的目录里，尽量保持现有结构不乱。

常见内容包括：

- 课件
- 作业
- 试卷
- 笔记
- 复习整理

如果文件里有个人隐私、学号、姓名、联系方式等内容，先处理掉再提交。

## 3. 本地检查

提交前简单看一遍：

- 目录放对了没有
- 文件名够不够清楚
- 有没有临时文件、缓存文件、重复文件
- 有没有误带个人信息

## 4. 提交 commit

建议一个 commit 对应一组相关资料，信息写得直接一点。

示例：

- `Add calculus final review notes`
- `Add database midterm papers`
- `Update course README for chapter 3 materials`

## 5. 提交 PR

请按标准 Pull Request 流程提交到 `main`。

PR 标题建议直接说明课程和内容，例如：

- `数据库：补充期末复习资料`
- `数据结构：新增第 4 章作业参考`

PR 描述建议包含：

- 变更内容
- 涉及课程
- 目录位置
- 资料来源
- 是否脱敏
- 是否有后续补充计划

## 6. 合并前自查

合并前再确认一次：

- 变更只包含你想提交的内容
- 没有混入无关格式化
- 没有把不该公开的内容带进来

## 一个简单模板

```text
标题：课程名 + 资料类型

内容：
- 新增了什么
- 放在哪个目录
- 来源是什么
- 是否已脱敏
```
