# 2.1 线性表及其基本运算

# 线性表的定义和特点

• 定义  $n (\geq 0)$  个数据元素的有限序列,记作

$$
(k _ {0}, k _ {1}, \dots , k _ {i}, \dots , k _ {n - 1})
$$

$k_{i}$  是表中数据元素,  $n$  是表长度。

• 原则上讲，线性表中表元素的数据类型可以不相同，但采用的存储表示可能会对其有限制。

# 线性表的特点

- 除第一个元素外，其它每一个元素有一个且仅有一个直接前驱。

- 除最后一个元素外，其它每一个元素有一个且仅有一个直接后继。

- 计算表的长度  $n$ ;

从左到右（或从右到左）遍历表的元素；

- 访问第  $i$  个元素,  $0 \leq i < n$ ;

- 将新值赋予第  $i$  个元素,  $0 \leq i < n$ ;

- 将新元素插入第  $i$  个位置,  $0 \leq i < n$ , 使原来的第  $i, i + 1, \ldots, n - 1$  个元素变为第  $i + 1, i + 2, \ldots, n$  个元素;

- 三对角矩阵中除主对角线及在主对角线上下最临近的两条对角线上的元素外，所有其它元素均为0，总共有  $3n - 2$  个非零元素。

- 将三对角矩阵  $\mathrm{A}$  中三条对角线上的元素按行存放在一维数组  $\mathrm{B}$  中, 且  $a_{00}$  存放于  $\mathrm{B}[0]$  。

- 在一维数组 B 中 A[i][j] 在第 i 行（i>0），它前面有  $3^{*}i - 1$  个非零元素，在本行中第 j 列前面有  $j - i + 1$  个（因为本行非零元素从  $j = i - 1$  开始，j 每增加 1，前面的元素数增加 1，因此偏移量为  $j - (i - 1) = j - i + 1$ ），所以元素 A[i][j] 在 B 中位置为  $k = 2^{*}i + j$ 。验证对于  $i = 0$  也成立。

- 若已知三对角矩阵中某元素 \(A[i][j]\) 在数组 \(\mathrm{B}[\] 存放于第 \(k\) 个位置, 则有

$$
\begin{array}{l} i = \left\lfloor (k + 1) / 3 \right\rfloor \\ j = k - 2 ^ {*} i \\ \end{array}
$$

例如, 当  $k = 8$  时,

$$
i = \left\lfloor (8 + 1) / 3 \right\rfloor = 3, j = 8 - 2 * 3 = 2
$$

当  $k = 10$  时,

$$
i = \left\lfloor (1 0 + 1) / 3 \right\rfloor = 3, j = 1 0 - 2 ^ {*} 3 = 4
$$

# 稀疏矩阵的定义

设矩阵  $\mathbf{A}_{m \times n}$  中有  $t$  个非零元素, 若  $t$  远远小于矩阵元素的总数  $m \times n$ , 则称矩阵  $\mathbf{A}$  为稀疏矩阵。

- 为节省存储空间，应只存储非零元素。

- 非零元素的分布一般没有规律，应在存储非零元素时，同时存储该非零元素的行下标 row、列下标 col、值 value。

- 每一个非零元素由一个三元组唯一确定：（行号 row, 列号 col, 值 value）

# 稀疏矩阵转置算法思想

- 如此，可以保证转置矩阵的三元组表行号从小到大，同一行列号从小到大

# 2.4 线性表的顺序表示——链表

# 为什么学习链表来表示线性表?

• 用数组或顺序映射表示线性表，相邻元素存储地址的间距是常数。因此，可用  $O(1)$  的时间访问任何元素。但是，删除或往其中插入元素将导致表中其它元素的移动，代价太高  $O(n)$  。

• 链表表示可以有效地解决上述数据移动问题。在链表表示中，表的数据元素可存放在存储器中的任何位置，每一个元素都与其下一个元素的地址（指针）相关联。

# 循环链表 (Circular List)

- 循环链表是单链表的变形。

- 循环链表的最后一个结点的 link 指针不为 NULL，而是指向表的前端。

- 为简化操作，在循环链表中往往加入表头结点。

- 循环链表的特点是：只要知道表中某一结点的地址，就可搜寻到所有其它结点的地址。

- 检查指针 current 是否到达链表的链尾时:

** 不是判断是否 current->link==NULL;

** 而是判断是否 current->link==first。

与单链表相比，循环链表在此场景下的优势

遍历终止条件更统一：

- 通过头节点的  $\exp = -1$  判断是否遍历完所有项，无需检查 NULL

首尾访问更方便:

从任意节点出发都能遍历整个链表，适合多项式频繁首尾操作的场景

- 结构更紧凑:

- 无需额外标记尾节点，通过头节点即可定位（尾节点的 link 指向头节点）

在单链表中，搜索一个指定结点的后继结点非常方便。

- 只要该结点的 link 域的内容不为空，就可通过 link 域找到该结点的后继结点地址。

在单链表中，搜索一个指定结点的前驱结点非常不容易。

必须从链头开始，沿 link 链顺序检测，直到某一结点的后继结点为该指定结点，则此结点即为该指定结点的前驱结点。

在一个应用问题中，经常要求检测指针向前驱和后继方向移动。

需要保证移动的时间复杂度达到最小。

# 2.5 进阶导读

- 线性表是一种最简单且最常用的数据结构，也是软件设计中最基础的数据结构。

- 用顺序方法存储的线性表称为顺序表，当线性表中很少做插入和删除操作，线性表的长度变化不大，易于事先确定其大小时，可以采用顺序表作为存储结构。

- 用链接方法存储的线性表称为线性链表，当线性表的长度变化较大，难以估计其存储规模，另外对线性表频繁进行插入和删除操作时，则采用链表作为存储结构可能会更好一些。

- 除动态链表外，还有静态链表。

- 在  $\mathrm{C} + +$  的 STL 库中提供关于线性表的几种类库, 如 vector、list 等。

# 静态链表

为数组中每一个元素附加一个链接指针。

× 允许不改变各元素的物理位置，只要重新链接就能够改变这些元素的逻辑顺序。

利用数组定义，在整个运算过程中存储空间的大小不会变化——静态链表。

每个结点由两个数据成员构成;

$\times$  data 域存储数据

$\times$ link 域存放链接指针

所有结点形成一个结点数组;

可带有表头结点。
