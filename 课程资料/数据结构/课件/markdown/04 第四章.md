# - 进栈操作

- 设栈的最后允许存放位置为 MAXN-1，首先判断栈是否已满。

如果栈顶指针 top=MAXN-1（栈已满），则栈中所有位置均已使用，若再有新元素进栈，将发生溢出，没有空间可以再利用，程序转入出错处理。

如果 top<MAXN-1，则先让栈顶指针进 1，指到当前可加入新元素的位置，再按栈顶指针所指位置将新元素插入，该元素成为新的栈顶元素。

- 当栈满时要发生溢出，程序报错并终止运行。

- 为避免这种情况，需要为栈设立一个足够大的空间。如果空间设置过大，而栈中实际只有几个元素，是一种空间浪费。

- 当程序只需要一个栈时，溢出问题不算太大。

- 当需要两个栈时，可定义一个足够大的栈空间。

# 双栈共享一个栈空间

![](file://~/MinerU/04 第四章.pptx-27bf11c6-16a7-4315-8123-c0f1d4ca3ffa/images/2974d087d9e487017e9723fadd92538a2231f02ef6344868e2d4d7e77cf641e7.jpg)


两个栈共享一个数组空间 V[maxSize]。

- 设立栈顶指针数组 t[2] 和栈底指针数组 b[2]，t[i] 和 b[i] 分别指示第 i 栈的栈顶与栈底。

• 初始  $t[0] = b[0] = -1, t[1] = b[1] = \text{maxSize}$ 。

栈满条件： $t[0] + 1 == t[1] // 栈顶指针相遇$

栈空条件： $t[0] = b[0]$  或  $t[1] = b[1]$  // 栈顶指针退到栈底

# - 当程序中同时存在几个栈，问题较复杂。

- 各个栈所需空间在运行中动态变化。

- 如果给几个栈分配同样大小的空间，可能在实际运行中，有的栈膨胀得快，很快就产生溢出。

- 其它栈可能还有许多空闲的空间，必须调整栈的空间，防止栈的溢出。

- 在插入时元素的移动量非常大，因而代价较高。

- 特别是当整个存储空间即将充满时，问题更加严重。

# 多栈处理栈浮动技术

-  $n$  栈共享一个数组空间  $V[m]$

• 设立栈顶指针数组  $t[n + 1]$  和栈底指针数组  $b[n + 1]$

-  $t[i]$  和  $b[i]$  分别指示第  $i$  个栈的栈顶与栈底,  $b[n]$  作为控制量, 指到数组最高下标。

- 各栈初始分配空间  $s = \lfloor m / n \rfloor$

• 指针初始值  $t[0] = b[0] = -1, b[n] = m - 1$ 
[ t[i] = b[i] = b[i - 1] + s, i = 1, 2, \dots, n - 1 ]

链式栈的栈顶在链头;

- 链式栈的栈顶设在链头。

# 一、两种表达式的定义与差异

# 1.中缀表达式 (Infix Expression)

定义：人类习惯的书写形式，运算符位于两个操作数中间

- 示例：a + b * c - (d + e)、3 + 4 / 2

- 特点：包含运算符优先级（如  $* > +$  ）和括号，存在计算歧义（计算机需额外判断顺序）

# 2.后缀表达式(PostfixExpression/逆波兰表达式)

定义：运算符位于两个操作数之后，无括号

示例：abc\*+de+-、342/+

- 特点：无歧义，计算顺序通过运算符位置直接确定（计算机可通过栈线性处理）

# 将中缀表示转为后缀表示的意义

# 1. 消除计算歧义，简化计算机处理

- 中缀表达式需判断“优先级”和“括号”，逻辑复杂；后缀表达式无需判断，仅需“线性遍历 + 栈操作”

例：中缀  $a + b*c$  需先算  $b*c$  ，后缀  $abc*+$  直接按顺序计算

# 2. 适配栈式计算硬件/软件

- 栈是计算机基础数据结构（CPU栈寄存器、虚拟机），后缀表达式的计算天然依赖栈

- 编译器、计算器、脚本引擎均需先将中缀转后缀，再执行计算

# 3. 提升计算效率

- 转换后计算时间复杂度为  $0(n)$  (  $n$  为表达式长度), 内存开销仅一个操作数栈

# 将中缀表示转为后缀表示的进阶算法

- 操作符栈初始化,将结束符(6)$' 进栈。然后读入中缀表达式字符流的首字符。

• 重复读入下一字符 ch，直到遇到 ‘\0’。

- 若 ch 是操作数直接输出，读入下一个字符。

- 若 ch 是 ‘(’，则入栈。

- 若 ch 是运算符  $\left(^{+}, ^{-}, ^{*}, ^{\land}\right)$ , 则将它的栈外优先级 (OSP) 与栈顶字符的栈内优先级 (ISP) 进行比较。如果当前的运算符的 OSP 小于等于栈顶字符的 ISP, 那么退出栈顶字符并输出。这样的过程一直进行到当前的运算符的 OSP 大于栈顶字符 ISP 为止, 然后让当前的运算符进栈。

- 若当前的字符是 ‘)’，则从栈中依次退出运算符并输出，直到 ‘(’为止，然后退出 ‘(’ ，但不输出 ‘(’。

- 若当前的字符是‘\0’，则从栈中依次退出所有运算符并输出，直到‘$’为止，‘$’不退栈，也不输出。

- 出队时，先将下标为 front 的元素取出，队头指针再进一 front=front+1。

队头指针指示实际队头所在位置

- 队空时再出队将队空处理。

# 双向队列

- 允许在两端进行插入和删除的线性表称为双向队列。双向队列没有队首和队尾之分，可用两个指针 left 和 right 分别指向双向队列中左端和右端结点。

- 双向队列空的标志为 left>right。

- 为了使得左右两端在一开始都能进行插入，取中间位置  $m = (MAXN - 1) / 2$  ，并置 left=m+1, right=m 作为双向队列的初始状态。

# 示例：

假设 MAXN = 7 (数组下标 0~6), 初始  $m = (7 - 1) / 2 = 3$ , 故 left = 4、right = 3 (空队列)。

在左端插入元素A：

left--  $\rightarrow$  left  $= 3$

数组arr[3] = A;

- 此时队列状态：left = 3、right = 3（left == right，队列中有1个元素）。

若继续在左端插入B：

left--  $\rightarrow$  left  $= 2$

arr[2] = B;

- 状态：left = 2、right = 3（队列中有2个元素，元素范围arr[2]~arr[3]）。
