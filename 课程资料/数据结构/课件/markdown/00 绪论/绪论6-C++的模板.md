# 函数模板

- 思考：如果重载的函数，其解决问题的逻辑是一致的、函数体语句相同，只是处理的数据类型不同，那么写多个相同的函数体，是重复劳动，而且还可能因为代码的冗余造成不一致性。

- 解决方案：使用模板

# 注意

- 一个函数模板并非自动可以处理所有类型的数据

- 只有能够进行函数模板中运算的类型，可以作为类型实参

- 自定义的类，需要重载模板中的运算符，才能作为类型实参

# 类模板

# 类模板的作用

- 使用类模板使用户可以为类声明一种模式，使得类中的某些数据成员、某些成员函数的参数、某些成员函数的返回值，能取任意类型（包括基本类型的和用户自定义类型）。

# 类模板的声明

- 类模板:

```txt
template <模板参数表>
class 类名
{类成员声明};
```

- 如果需要在类模板以外定义其成员函数，则要采用以下的形式：

□ template < 模板参数表 >

□ 类型名 类名 <模板参数标识符列表 >:: 函数名 (参数表)

- 模板参数表中参数可以声明为该模板类的友元类

- 可通过 typedef 或者 using 对实例化的类模板定义别名

# 模板的默认实参

类似于函数形参可有默认实参，函数/类模板可有默认模板实参：

template<class T>

class Store{//类模板：实现对任意类型数据进行存取

private:

T item; // item 用于存放任意类型的数据

public:

friend T; // 将参数类型 T 声明为友元类。例如：当 T=Student 时，Student 结构体（或其对象）

可以直接访问 Store<Student> 的私有成员 item 和 haveValue

Store();

T &getElem(); // 提取数据函数，返回 item 的引用

void putElem(const T &x); // 存入数据函数，接收 T 类型的常量引用 x，将其赋值给 item };

# 例 类模板示例

template<class T> // 模板声明前缀——由于Store是类模板，其所有成员函数的实现都必须以该前缀开头，表明后续代码属于“针对任意类型T的Store类成员”

Store<T>::Store(): haveValue(false) {}

template<class T>

T &Store<T>::getElem() {

//如试图提取未初始化的数据，则终止程序

if (!haveValue) {

cout << "No item present!" << endl;

exit(1); // 使程序完全退出，返回到操作系统。

}

return item; //返回item中存放的数据

}

template<class T>

void Store<T>::putElem(const T &x) {

// 将haveValue置为true，表示item中已存入数值

haveValue = true;

item = x; // 将 x 值存入 item。当 T 为类，C++ 编译器会自动生成一个默认赋值运算符进行逐成员赋值

}
