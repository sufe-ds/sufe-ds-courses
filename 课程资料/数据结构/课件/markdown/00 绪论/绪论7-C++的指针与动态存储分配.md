# 指向常量的指针

- 不能通过指向常量的指针改变所指对象的值，但指针本身可以改变，可以指向另外的对象。

- 例

```c
int a;  
const int *p1 = &a; //p1是指向常量的指针  
int b;  
p1 = &b; //正确，p1本身的值可以改变  
*p1 = 1; //编译时出错，不能通过p1改变所指的对象
```

# 指针类型的常量

- 若声明指针常量, 则指针本身的值不能被改变。

- 例

int a;

int * const p2 = &a;

int b;

p2 = &b; // 错误, p2 是指针常量, 值不能改变

# 动态内存分配

# 动态申请内存操作符 new

- new 是用于动态分配内存的操作符。它不仅能在堆上开辟指定大小的内存空间，还会自动调用对象的构造函数（如果分配的是类对象）。

- 分配单个对象

• 格式：指针类型 指针变量名 = new 数据类型（初始化值）; int* p = new int(10); // 分配一个 int 类型的内存，初始化为 10, p 指向该内存

- 分配数组

- 格式：指针类型 指针变量名 = new 数据类型「数组大小」; int* arr = new int[5]; // 分配一个包含 5 个 int 类型元素的数组，arr 指向数组首地址

# 释放内存操作符 delete

- delete 指针变量名

• 例如: delete p; // 释放指针 p 所指向的由 new 分配的单个 int 类型内存。

- delete[] 指针变量名

例如：delete[] arr; // 释放 arr 指向的由 new 分配的 int 数组内存

# 注意事项

- new 和 delete 必须成对使用，否则会导致内存泄漏。

- 内存泄漏是指程序在动态分配内存后，无法释放已分配的内存空间，导致系统内存被逐渐耗尽，影响程序的性能甚至导致程序崩溃。

- 不能使用 delete 释放不是由 new 分配的内存，也不能对同一个指针使用多次 delete。

- 当 new 分配内存失败时，在现代 C++ 中会抛出 std::bad_alloc 异常（早期可能返回空指针）
